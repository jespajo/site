principles for decent javascript written by someone who is very much still learning:

  • javascript is a beautiful language
  • code should be as concise as possible BUT
  • code should also be readable by humans. including documentation where necessary. the dream is self-documenting code: no need for documentation because the code is a natural expression of human ideas
  • use vanilla js rather than libraries where possible, even if it means code is "longer" (actually probably not true if you count the library the user has to load; also libraries/frameworks change but the underlying language generally stays the same and you have to master that)
  • functional programming
  • move slow and fix things
